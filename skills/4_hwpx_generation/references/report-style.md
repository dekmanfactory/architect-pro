## 1. 문서 구조

보고서는 공문(기안문)과 달리 다음 구조를 갖는다:

```
1쪽: 표지
  ├─ 상단: 기관 로고/장식 테이블
  ├─ 중앙: 보고서 제목 (30pt, 25pt HY헤드라인M, 가운데 정렬)
  └─ 하단: 작성일 (25pt HY헤드라인M)

2쪽: 목차
  ├─ 장식 배너 테이블 (색상 그라데이션)
  ├─ "목  차" 타이틀 (28pt HY헤드라인M)
  └─ 목차 항목 (로마 숫자 Ⅰ~Ⅴ + 제목 + 페이지)

3쪽~: 본문
  ├─ 결재란 테이블 (3x3, 페이지 상단)
  ├─ 보고서 제목 (22pt HY헤드라인M, 가운데 정렬)
  ├─ 섹션 구분 바 (1x3 색상 테이블)
  └─ 본문 계층 구조 (□ → ○ → ― → ※)
```

## 4. 항목 기호 체계

보고서에서는 공문서 번호 체계(1. 가. 1) …)가 아닌 **기호 체계**를 사용한다:

```
1단계:  □    (HY헤드라인M 16pt, 문단 위 15)
2단계:  ○    (휴먼명조 15pt, 문단 위 10)
3단계:  ―    (휴먼명조 15pt, 문단 위 6)
4단계:  ※    (한양중고딕 13pt, 문단 위 3)
```

### 작성 예시

```
 □ 2026년 AI 기술 도입 현황 분석 결과
  ○ 전년 대비 도입률 23% 증가
   ― 공공기관 도입률: 45.2% (전년 36.8%)
     ※ 교육청 및 산하기관 포함 수치
  ○ 주요 도입 분야
   ― 행정업무 자동화: 전체의 58%
   ― 민원 상담 챗봇: 전체의 27%
     ※ 나머지 15%는 데이터 분석 및 기타
```

### □ 기호의 이중 run 구조

XML 내부에서 □ 항목은 두 개의 run으로 구성된다:
- **run 1**: `□` 기호 자체 (13pt, 작은 크기)
- **run 2**: 본문 텍스트 (16pt, HY헤드라인M)

이 이중 구조는 python-hwpx 기본 API로 재현하기 어렵다. 템플릿 기반 접근을 권장하는 이유 중 하나이다.

## 5. 섹션 구분 바 테이블

보고서의 대단원(Ⅰ, Ⅱ, Ⅲ …)을 구분하는 디자인 요소. 1행 3열 테이블로 구성된다.

```
┌──────────┬──┬─────────────────────────────────┐
│    Ⅰ     │  │        추진 배경                  │
│ 남색 배경  │  │ 연회색 배경, 상하 회색 테두리      │
│ 흰색 글자  │  │                                  │
└──────────┴──┴─────────────────────────────────┘
```

### 셀 상세

| 셀 | 너비(HWPUNIT) | 내용 | 배경색 | 글자색 | 글꼴/크기 |
|----|-------------|------|--------|--------|----------|
| 0 | 3327 (~12mm) | 로마숫자 | #193AAA (남색) | #FFFFFF | 휴먼명조 17pt |
| 1 | 848 (~3mm) | (빈칸, 구분선) | 없음 | — | — |
| 2 | 43513 (~153mm) | 섹션 제목 | #F2F2F2 (연회색) | #000000 | HY헤드라인M 17pt |

전체 크기: 47688 × 2832 HWPUNIT (약 168mm × 10mm)

> 이 테이블은 python-hwpx 기본 API로 셀 배경색을 지정할 수 없으므로, **반드시 템플릿 기반 접근**으로 처리한다. 템플릿의 섹션 바를 유지하고 텍스트(로마숫자, 제목)만 치환하는 방식을 사용한다.

## 8. 결재란

본문 시작 페이지(3쪽)에 위치하는 3x3 테이블.

```
┌─────────────────────────────────────┐
│         (결재란 3x3 테이블)           │
│  크기: 48180 × 4766 HWPUNIT         │
│  위치: 페이지 상단                    │
│  pageBreak="1" (새 페이지 시작)       │
```

> 결재란의 구체적 셀 구성은 양식마다 다르다. 브라더 양식에서는 표지가 아닌 본문 시작 페이지에 결재란을 배치한다.

## 10. 템플릿 기반 활용 패턴

보고서의 디자인 요소(색상 배경, 섹션 바, 장식 테이블)는 python-hwpx 기본 API로 재현하기 어렵다. **`assets/report-template.hwpx`** 를 템플릿으로 사용하는 것을 강력히 권장한다.

### 워크플로우

```python
import shutil
from hwpx.document import HwpxDocument

# 1. 템플릿 복사
TEMPLATE = "/mnt/skills/user/hwpx/assets/report-template.hwpx"
shutil.copy(TEMPLATE, "/home/claude/report.hwpx")

# 2. 본문 텍스트 치환
doc = HwpxDocument.open("/home/claude/report.hwpx")
doc.replace_text_in_runs("브라더 공기관", "AI정책실")
doc.replace_text_in_runs("기본 보고서 양식", "2026년 AI 활용 현황 보고")
doc.replace_text_in_runs("2024. 5. 23.", "2026. 2. 13.")
doc.save("/home/claude/report.hwpx")
fix_hwpx_namespaces("/home/claude/report.hwpx")

# 3. 표 내부 포함 전체 치환 (섹션 바 제목 등)
from hwpx_replace import replace_all_text_in_hwpx
replace_all_text_in_hwpx(
    src_path="/home/claude/report.hwpx",
    dst_path="/home/claude/report_final.hwpx",
    replacements={
        "제 목": "AI 활용 현황 및 개선 방안",
        "헤드라인M 폰트 16포인트(문단 위 15)": "실제 본문 내용",
        "휴면명조 15포인트(문단위 10)": "실제 세부 내용",
        "휴면명조 15포인트(문단 위 6)": "실제 상세 내용",
        "중고딕 13포인트(문단 위 3)": "참고 사항",
    }
)
```

### 템플릿 내 치환 가능한 텍스트

| 원본 텍스트 | 위치 | 치환 대상 |
|------------|------|----------|
| `브라더 공기관` | 표지 1줄 | 기관명 |
| `기본 보고서 양식` | 표지 2줄 | 보고서 제목 |
| `2024. 5. 23.` | 표지 작성일 | 실제 작성일 |
| `제 목` | 본문 페이지 제목 | 보고서 제목 |
| `헤드라인M 폰트 16포인트(문단 위 15)` | □ 본문 | 1단계 내용 |
| `휴면명조 15포인트(문단위 10)` | ○ 본문 | 2단계 내용 |
| `휴면명조 15포인트(문단 위 6)` | ― 본문 | 3단계 내용 |
| `중고딕 13포인트(문단 위 3)` | ※ 주석 | 4단계 참조 |
